# 需求文档：需求进度追踪器文档标准化

## 基本信息

- **需求 ID**: req-20260329-222104
- **创建时间**: 2026-03-29T14:21:04.857Z
- **状态**: Draft → Review → Approved

## 概述

优化需求进度追踪器中 brainstorm 和 design 阶段的文档产出，实现产出位置、文档名称、格式的标准化和固定化，解决当前文档不统一、维护困难的问题。

## 用户故事

作为 ECC 项目维护者，我希望需求追踪器的各阶段能产出固定格式的文档，以便于文档管理和维护。

## 功能需求

### 专用 Agent

#### 描述
新增两个专用 Agent 负责文档撰写，使 Agent 职责单一化。

#### 验收标准
- [ ] requirement-doc-writer agent 存在且功能正常
- [ ] design-doc-writer agent 存在且功能正常
- [ ] Agent 只引用模板文件，不内置模板内容

### 文档模板标准化

#### 描述
文档模板统一存放在固定目录，使用统一格式。

#### 验收标准
- [ ] 模板目录为 requirements/templates/docs/
- [ ] 包含 requirement-doc-template.md
- [ ] 包含 high-level-design-template.md
- [ ] 包含 detailed-design-template.md

### 文档输出位置标准化

#### 描述
各阶段产出的文档存储在固定位置，便于查找和管理。

#### 验收标准
- [ ] 需求文档输出到 .requirements/docs/{id}/需求文档.md
- [ ] 概要设计输出到 .requirements/docs/{id}/概要设计.md
- [ ] 详细设计输出到 .requirements/docs/{id}/详细设计.md

### 工作流调整

#### 描述
移除 plan 阶段，优化 medium 模板的工作流。

#### 验收标准
- [ ] medium 模板移除 plan 阶段
- [ ] brainstorm 阶段调用 requirement-doc-writer
- [ ] design 阶段调用 design-doc-writer
- [ ] config.json5 中的工作流定义已更新

## 非功能需求

### 性能
- 文档生成时间 < 5s

### 安全性
- 无敏感信息存储

### 可用性
- 文档路径固定，易于查找
- 模板格式统一，易于维护

## 边界情况

- 如果模板文件不存在，Agent 应给出明确错误信息
- 如果输出目录创建失败，应回退并报告错误

## 依赖项

- Node.js 14+
- 文件系统读写权限

## 风险与假设

- **风险**: Agent 生成的文档内容可能不符合预期
  - **缓解措施**: 提供详细的模板和 prompt 约束

- **假设**: 用户会按照固定流程创建需求
