# 概要设计：需求进度追踪器文档标准化

## 基本信息

- **需求 ID**: req-20260329-222104
- **关联需求文档**: 需求文档.md
- **状态**: Draft → Review → Approved

## 1. 系统架构

### 1.1 整体架构

```
agents/
├── requirement-doc-writer.md    # 需求文档撰写 Agent
└── design-doc-writer.md         # 设计文档撰写 Agent

requirements/
└── templates/
    └── docs/                   # 文档模板目录
        ├── requirement-doc-template.md
        ├── high-level-design-template.md
        └── detailed-design-template.md

.requirements/
└── docs/
    └── {requirement-id}/        # 需求文档输出目录
        ├── 需求文档.md
        ├── 概要设计.md
        └── 详细设计.md
```

### 1.2 模块划分

| 模块名称 | 职责 | 依赖模块 |
|----------|------|----------|
| requirement-doc-writer | 需求文档生成 | 模板系统 |
| design-doc-writer | 设计文档生成 | 模板系统 |
| 模板系统 | 提供标准化模板 | 无 |
| 文档输出 | 管理文档存储位置 | 文件系统 |

## 2. 技术选型

### 2.1 技术栈

- Node.js (JavaScript)
- 文件系统 API (fs/promises)
- Markdown 格式

### 2.2 关键技术决策

- **模板外部化**: 模板与 Agent 分离，便于单独维护
- **路径固定化**: 统一使用 .requirements/docs/{id}/ 作为输出根目录
- **Agent 单一职责**: 每个 Agent 只负责一种文档类型

## 3. 数据模型

### 3.1 核心实体

- **Requirement**: 需求实体，包含 id、name、type、status、acceptanceCriteria 等字段
- **Progress**: 进度实体，包含 phases 数组，每个 phase 有 name、mode、agent、status
- **ProgressLog**: 日志实体，记录事件、时间和详情

### 3.2 实体关系图

```
Requirement 1 ---> * ProgressPhase
Requirement 1 ---> * ProgressLog
ProgressPhase 1 ---> 1 Agent
```

## 4. API 设计

### 4.1 核心接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| createRequirement | async | 内部函数 | 创建需求 |
| updateRequirementProgress | async | 内部函数 | 更新进度 |
| generatePhaseHandoff | async | 内部函数 | 生成交接文档 |

## 5. 系统交互流程

### 5.1 主要流程

1. 用户创建需求 → 生成需求 JSON 文件
2. brainstorm 阶段 → requirement-doc-writer 生成需求文档
3. design 阶段 → design-doc-writer 生成设计文档
4. implement 阶段 → tdd-guide 指导实现
5. 各阶段完成后 → 生成 handoff 文档

### 5.2 异常处理流程

- 模板文件不存在 → 抛出明确错误
- 输出目录创建失败 → 报错并回退
- Agent 执行失败 → 记录错误到 progressLog

## 6. 非功能设计

### 6.1 性能

- 文档生成操作应为同步或短暂异步，不阻塞主流程

### 6.2 安全

- 无敏感数据处理
- 文件路径经过验证，防止路径遍历

### 6.3 扩展性

- 模板系统可独立扩展新文档类型
- Agent 可配置指向不同模板

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Agent 生成内容不符合预期 | 中 | 提供详细 prompt 和模板约束 |
| 模板格式变化导致兼容性问题 | 中 | 版本化管理模板 |
| 文档输出目录被误删 | 低 | 可重新生成 |

## 8. 部署方案

- 无需特殊部署
- 文档模板随项目一起版本控制
- Agent 文件位于 agents/ 目录
